### Runtime Screenshot Preservation Policy

Runtime screenshots and walkthroughs are dynamic proof artifacts. Their file hashes prove mirror/copy integrity for the same captured artifact, not visual sameness across separate browser/native captures.

For existing-surface preservation, reviewers should require a semantic preservation gate instead of byte-identical runtime PNGs. The gate should verify the protected route is present, the protected UI state is reachable, core labels and sections remain visible, screenshots are fresh/nonblank, both required themes are represented when applicable, and deterministic source or manifest artifacts match where true identity is required.

Do not copy old screenshots into a current evidence bundle just to force matching hashes. That is screenshot laundering and is a gate failure. If a visual change is intentional, capture fresh evidence and record the decision rather than pretending the new runtime image is byte-identical to the old one.

## Agent Routing

The system can balance work across multiple AI agents. Each agent has a credit pool, strengths, and a CLI invocation. The orchestrator picks the best agent for each task based on `task_type` and remaining credits.

**Note:** Monthly credit budgets are disabled. The platform runs on flat subscription plans — the admin monitors usage directly. The budgets below are set intentionally high (10000) to prevent any throttling. Metering still tracks invocation counts for observability, but does NOT enforce limits or trigger budget-based routing decisions.

```yaml
agent_routing:
  # agent_mode controls how work is routed across agents:
  #   chat_native (DEFAULT) — single-host mode. The orchestrator and every executor
  #     run on whichever CLI is hosting the chat (Claude Code or Codex). Detection
  #     priority: explicit host_agent below → CLAUDECODE / CODEX_HOME env vars →
  #     defaults to claude. This is the right default — most users run inside one CLI.
  #   normal — cross-model mode. Uses the task_routing table below to route specific
  #     task types to specific agents (e.g. code_build → codex, visual_review → claude).
  #     Requires both CLIs configured and enabled. Strengthens gates with cross-model
  #     review but isn't required to run end-to-end.
  #   claude_fallback / codex_fallback — explicit single-agent overrides; route ALL
  #     work to the named agent regardless of host detection. Useful when one CLI is
  #     unavailable mid-run on a normal-mode setup.
  # Manage via `python3 scripts/set_agent_mode.py {mode}`.
  agent_mode: codex_fallback
  # host_agent (optional): explicitly declare which CLI hosts the orchestrator.
  # Leave blank to auto-detect via env vars. Set to "claude" or "codex" to override.
  # Used only when agent_mode is chat_native.
  host_agent: "codex"
  budget_based_routing: false
  orchestration_context_mode: tiered   # reversible: set to full to restore legacy broad-context Claude orchestration
  orchestration_context_packet_max_chars: 32000
  orchestration_context_expand_on: [phase_gate, ambiguous_decision, conflicting_evidence, operator_or_client_communication, system_anomaly, delivery_or_completion]
  agents:
    claude:
      cli: "claude -p --model claude-opus-4-7 --effort max"
      enabled: true
      strengths: [orchestration, planning, stakeholder-communication, visual-judgment, taste, multimodal-review]
      monthly_credit_budget: 10000
      priority: 1  # control-plane preference; worker routing is task-type driven
    codex:
      cli: "/opt/homebrew/bin/codex exec"
      enabled: true
      strengths: [code-review, bug-fixing, refactoring, test-generation, code-writing, debugging, execution-briefs, proof-manifests, evidence-audits, quality-checks]
      monthly_credit_budget: 10000
      priority: 2
    gemini:
      cli: "/opt/homebrew/bin/gemini"
      enabled: false
      strengths: [artifact-cleanup, receipt-cleanup, docs-cleanup, bounded-rewrites]
      monthly_credit_budget: 10000
      priority: 3
  task_routing:
    # Task type → preferred agent. Falls back only if the preferred agent is disabled/unavailable.
    planner: claude
    orchestrator: claude
    design: claude
    orchestration: claude
    creative_brief: codex      # execution-contract briefs default to Codex; route true stakeholder narrative/taste work as visual_review/design when Claude is needed
    verification_manifest_generate: codex   # preferred name: derives proof manifest from brief
    verification_manifest_execute: codex    # preferred name: executes proof manifest
    test_manifest_generate: codex           # legacy alias
    test_manifest_execute: codex            # legacy alias
    self_review: codex        # default worker review lane, including UI/Stitch surfaces
    quality_check: codex      # default QC lane, including UI/Stitch surfaces; use visual_review for Claude visual judgment
    visual_review: claude      # must be Claude — authoritative screenshot/design judgment gate
    visual_spec: claude
    visual_spec_review: claude
    artifact_polish_review: codex   # clean-room artifact consumption review by default; use visual_review/design for Claude taste calls
    credibility_gate: codex    # trust/claim audit + fresh-checkout verification
    drift_detection: codex
    vault_navigation: codex
    onboarding: claude
    simulation_rehearsal: codex
    project_change_control: codex      # deterministic amendment classification / scoped update work
    project_amendment: codex           # structured project-amendment artifact handling
    project_replan: codex              # plan rebaseline / roadmap reconciliation, not control-plane judgment
    plan_rebase: codex
    plan_rebaseline: codex
    plan_reconciliation: codex
    roadmap_reconciliation: codex
    architecture_decision: codex       # ADR/editing lane; route strategy_review/final_judgment to Claude when needed
    build: codex               # legacy alias for code_build
    code_build: codex          # codex builds all code (was: claude)
    code_review: codex         # code review stays on the premium code lane; hard gate/delivery reviews also force codex
    code_fix: codex
    test_generation: codex
    mcp_build: codex           # codex builds MCPs (was: claude)
    mcp_review: codex          # codex reviews/debugs it
    artifact_cleanup: codex    # route catch-all cleanup work to the premium code lane
    receipt_cleanup: codex
    docs_cleanup: codex
    brief_remediation: codex    # targeted brief/parser/requirements cleanup; not net-new creative brief generation
    evidence_cleanup: codex     # evidence path, QC report, and citation hygiene remediation
    gate_remediation: codex     # legacy/control-plane only; orchestrator must not create project tickets of this type
    general: codex             # default non-specialized work lane
  routing_override_tags: []     # contract tags enforce prompts/gates, not Claude routing; task_type decides agent
```
