# AI Disclosure Footer

## Purpose

This disclosure ensures transparency and compliance with emerging AI disclosure regulations. It informs recipients that communications are AI-generated and provides avenues for questions, the full Terms of Service, and the Privacy Policy.

This footer is **mandatory** on all outbound communications, including but not limited to:

- Cold outreach emails
- Client update emails
- Delivered work product
- Proposals and quotes
- Follow-up messages
- Any automated notification sent to an external party

## Standard Footer

The following block must be appended to every outbound email, separated from the message body by a horizontal rule:

```
---
This message was generated by an AI agent operated by [LLC FULL LEGAL NAME].
No human reviewed this message prior to sending.
For questions, contact [CONTACT EMAIL].
Terms of Service: [TERMS OF SERVICE URL]
Privacy Policy: [PRIVACY POLICY URL]
```

## Compact Footer (Character-Limited Contexts)

For contexts with character limits (e.g., SMS, social media DMs), use this abbreviated version:

```
---
AI-generated by [LLC SHORT NAME]. Info: [CONTACT EMAIL] | Terms: [SHORT URL]
```

## Deliverable Watermark

For documents, reports, and other deliverables sent to clients, include the following notice on the first page or in the document header:

```
This document was generated by an AI system operated by [LLC FULL LEGAL NAME].
It has not been reviewed by a human unless otherwise noted.
Generated: [DATE] | Reference: [PROJECT/TASK ID]
```

## Implementation Notes

- The agent must include this footer **before** sending any external message. This is a hard rule with no override.
- If the footer cannot be appended for technical reasons, the message must not be sent, and the issue must be escalated.
- The URLs for Terms of Service and Privacy Policy must be kept current. Update them in the agent configuration if they change.
